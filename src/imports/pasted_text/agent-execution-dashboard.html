卡片：白底 + 1px 浅灰边框 + 轻微阴影，hover 时阴影加深 + 微上移 - 字体：中文用 Noto Sans SC，英文/数字用 DM Sans，大数字 28px bold，标签 12-13px - 不要使用 Inter / Roboto / Arial 等泛用字体 - 所有进度条和堆叠条使用圆角，高度 3-10px - 动画：页面加载时卡片依次 fadeIn + translateY 入场，stagger delay 50ms ### 四、交互要求 - 所有 Tab 组件支持点击切换 active 状态 - 场景 Toggle 栏支持多选/全选逻辑（详见第一部分说明） - 筛选器变化时更新页面副标题显示当前筛选条件 - 重置按钮一键清空所有筛选，场景回到「全部」 - Tooltip 在 hover 时显示指标解释 ### 五、关键设计原则 1. **每个核心指标必须包含「全局值 + 单/多智能体拆分」双层结构**——这是 toB 系统看板与简单 dashboard 的核心区别 2. **应用场景必须是常驻可见的 toggle chip 栏**，不要藏在下拉菜单里，因为运维人员需要频繁在不同场景间切换对比 3. **质量模块必须包含「全局概览 + 按场景对比表」双层视图**——全局数据用于总览，对比表用于定位具体场景的质量洼地 4. **瓶颈模块必须标注智能体所属场景和类型**——便于快速归因，判断问题出在单智能体还是协作链路 5. **整体信息密度要高**，但通过清晰的视觉层级（标题 > 大数字 > 次要信息 > 辅助文字）让用户可以快速扫描 请直接输出完整的 HTML 单文件代码，不要分拆 CSS/JS 文件
